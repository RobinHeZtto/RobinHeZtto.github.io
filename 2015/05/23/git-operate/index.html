git操作总结

发表于 2015-05-23   |  

基本操作

  1. 配置
    Usage: git config –global|–system|–local|–list
    $ git config --global user.name [username]
    $ git config --global user.email [email]
    $ git config --global color.ui true

  2. 初始化版本库
    $ git init

  3. 暂存
    $ git add .
    $ git add -u 暂存已跟踪文件
    $ git add -A 跟踪文件并暂存
    $ git add -p 选择性暂存文件

  4. 状态查看
    $ git status
    $ git status -s 精简显示

  5. 比较
    工作区与暂存区比较:
    $ git diff
    暂存区与HEAD(版本库当前分支)比较:
    $ git diff --staged
    工作,暂存区(未提交修改)与HEAD(版本库当前分支)比较:
    $ git diff HEAD

  6. 撤销
    撤销工作区修改:
    $ git checkout .
    该命令会将暂存区全部文件或指定文件替换工作区对应文件
    $ git checkout -- <file>

    撤销工作区,暂存区修改:
    $ git checkout HEAD .
    $ git checkout HEAD <file>
    该命令会将HEAD所指向的master分支全部或指定文件替换到工作区,暂存区,

    暂存区域撤销:
    $ git reset HEAD

  7. 移除
    暂存区移除:
    $ git rm --cached <file>

  8. 日志查看
    git log

  9. 其他

  10. 修改远程仓库url:
    git remote set-url origin new-url

实例

Git global setup
git config –global user.name “HE”
git config –global user.email “robin.he.ztto@gmail.com”

Create a new repository
git clone xxx.git
cd fota
touch README.md
git add README.md
git commit -m “add README”
git push -u origin master

Existing folder
cd existing_folder
git init
git remote add origin xxx.git
git add .
git commit -m “Initial commit”
git push -u origin master

Existing Git repository
cd existing_repo
git remote rename origin old-origin
git remote add origin xxx.git
git push -u origin –all
git push -u origin –tags

Android Binder之Binder Driver

发表于 2016-09-06   |  

binder驱动源码位于drivers/staging/android/下，主要包括binder.c与uapi/binder.h

数据结构

在分析驱动之前，先了解binder驱动中所定义的数据结构.binder的数据结构大概可以分成二类看，一类是内部使用的，在binder.c中定义，另一类是内外部使用，在binder.h中定义.

binder_node

binder_node在binder.c中定义，属于binder driver内部数据结构，它是binder实体在内核中的描述.

  1. union {
    struct rb_node rb_node;
    struct hlist_node dead_node;
    };
    binder_node未销毁时通过rb_node链接到proc->nodes红黑树上.
    binder_node销毁时通过dead_node加入到死亡的hash list上.
  2. struct binder_proc *proc;　binder实体(服务)所在的宿主进程.
  3. struct hlist_head refs; binder的引用hash list,所有该binder实体的引用都保存在该hash list中.
  4. binder_uintptr_t ptr; 描述用户空间的service組件，Binder实体对应的Server在用户空间的本地Binder的引用
  5. binder_uintptr_t cookie; 描述用户空间的service組件，Binder实体对应的Server在用户空间的本地Binder地址

binder_ref

binder_ref在binder.c中定义，属于binder driver内部数据结构，它是binder引用(Client组件)在内核中的描述.

  1. struct rb_node rb_node_desc; 以句柄值作为关键字关联到binder_proc->refs_by_desc红黑树
  2. struct rb_node rb_node_node;　以binder实体对象地址作为关键字关联到binder_proc->refs_by_node红黑树
  3. struct hlist_node node_entry; binder_node的引用节点，关联到binder_node->refs
  4. struct binder_proc *proc; 引用所在的宿主进程.
  5. struct binder_node *node; 引用所对应的binder_node实体.
  6. uint32_t desc; Binder引用的句柄值，Binder驱动为binder引用分配的一个唯一的int型整数（进程范围内唯一）

binder_proc

binder_proc在binder.c中定义，属于binder driver内部数据结构，它是使用binder通信的进程在在内核中的描述.

  1. struct hlist_node proc_node; 全局哈希链表binder_procs的节点，所有binder_proc都被加入到该链表
  2. struct rb_root threads; binder_thread红黑树，关联binder_thread->rb_node
  3. struct rb_root nodes; binder_node红黑树，关联binder_node->rb_node
  4. struct rb_root refs_by_desc;　以binder_ref描述句柄为关键字的红黑树，关联binder_ref->rb_node_desc
  5. struct rb_root refs_by_node; 以binder_ref关联的实体地址为关键字的红黑树，关联binder_ref->rb_node
  6. int pid;　进程id
  7. struct vm_area_struct *vma; 进程虚拟地址空间
  8. struct task_struct *tsk;　进程task_struct控制块
  9. void *buffer;　进程在内核虚拟内存中的起始地址
  10. ptrdiff_t user_buffer_offset;　内核虚拟地址和用户虚拟地址之间的差值
  11. struct list_head todo;　进程的待处理事务队列
  12. wait_queue_head_t wait;　等待队列

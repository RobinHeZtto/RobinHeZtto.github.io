Android之Zygote-应用进程创建

发表于 2016-11-28   |  

Android上的应用程序通过Zygote来启动，应用由各种组件(activity,service,broadcastReceiver,contentProvider)组合，进程则充当运行这些组件的容器，在启动应用的某个组件之前，首先必须向zygote请求新建该应用的进程，此篇博客主要分析Zygote创建应用进程的过程。相关源码在以下文件中：

  • frameworks/base/core/java/android/os/Process.java
  • frameworks/base/core/java/com/android/internal/os/ZygoteInit.java
  • frameworks/base/core/java/com/android/internal/os/ZygoteConnection.java
  • frameworks/base/core/java/com/android/internal/os/Zygote.java
  • frameworks/base/core/jni/com_android_internal_os_Zygote.cpp

system_server通过socket与zygote进行通信，下面从client与server二方面来分析该流程。

Client Request

创建进程的请求由client端system_server的AMS通过localSocket向Zygote发起,下面是请求创建进程接口Process.start()的实现：

Process.start()请求zygote创建进程并执行参数中传递的processClass类(“android.app.ActivityThread”)的main方法，并指定进程名niceName，uid，gid等进程相关信息。

startViaZygote()的主要工作是创进程启动参数argsForZygote，并通过zygoteSendArgsAndGetResult发送给zygote请求fork子进程。
zygoteSendArgsAndGetResult()的第一个参数类型是ZygoteState，ZygoteState是Process.java中的一个静态内部类，负责与zygote socket通信细节。在分析zygoteSendArgsAndGetResult之前先分析一下ZygoteState。

zygoteSendArgsAndGetResult的第一个参数为openZygoteSocketIfNeeded(abi),即通过abi选择对应的ZygoteState(分别connect /dev/socket/zygote或/dev/socket/zygote_secondary)。

回到zygoteSendArgsAndGetResult方法，继续分析。

zygoteSendArgsAndGetResult向zygote发送创建进程请求及参数，并等待创建结束返回pid．client端的工作结束，下面分析server端的处理．

Server Handle

通过在init.${ro.zygote}.rc中的定义，init进程创建出zygote socket。

下面从ZygoteInit.main()开始分析在Zygote服务端处理创建进程的流程．

ZygoteInit.main()中首先通过registerZygoteSocket创建服务端LocalServerSocket，然后在runSelectLoop()中循环等待处理客户端连接并处理．

registerZygoteSocket()的主要作用创建Zygote服务端sServerSocket，并设置/dev/socket/zygote*的文件描述符。

在监听到与客户端连接的socket有数据到来是，调用ZygoteConnection.runOnce()处理客户端的数据

在runOnce中读取并解析客户端发送的启动参数，并调用Zygote.forkAndSpecialize()方法fork子进程。

forkAndSpecialize中直接调用native方法nativeForkAndSpecialize，nativeForkAndSpecialize在register_com_android_internal_os_Zygote在AndroidRuntime.start()中通过startReg注册。

nativeForkAndSpecialize映射到jni方法com_android_internal_os_Zygote_nativeForkAndSpecialize，

最终ForkAndSpecializeCommon中通过fork创建出子进程并返回，接着在子进程(pid == 0)中继续执行handleChildProc
